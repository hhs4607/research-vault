# Deep Dive

Read the note first, then search the vault for nearby context. If a sequential-thinking tool is available, use it; otherwise reason directly.

## Workflow

1. Read the target note.
2. Search for related notes, tags, and linked concepts.
3. Identify 2-4 promising threads:
   - hidden assumptions
   - concepts not fully explained
   - claims worth stress-testing
   - connections to Paper-1 through Paper-4
4. Discuss the most promising thread with the user.
5. When the discussion is mature enough, append a `## Deep Dive (YYYY-MM-DD)` section to the original note.
6. Update tags only if the deep dive genuinely broadened the note's scope.

## Append Style

- Preserve the explanatory detail from the discussion.
- Use headings per topic.
- End with a concise `### Key Insight` blockquote when helpful.

## Guardrails

- Do not merely restate the note.
- Keep the appended section anchored to the user's actual questions and the vault context.
